## （三）尚缺少的条件及解决方法

### 缺少条件1：Android构建环境
**现状**：Unity Hub中尚未安装Android Build Support模块
**影响**：无法打包APK/ABB部署到Rokid AR Lite设备
**解决方法**：
- 在Unity Hub → Installs → Unity 6000.4.0f1 → Add Modules中安装：
  - ☑️ Android Build Support（必须）
  - ☑️ Android SDK & NDK Tools（必须）
  - ☑️ OpenJDK（必须）
- 预计安装时间：模块总计约5-8GB磁盘空间
- 配置Player Settings：Scripting Backend=IL2CPP, Target Architectures=ARM64

### 缺少条件2：AR SDK集成未完成
**现状**：UXR 2.0或OpenXR插件尚未正式集成到项目中
**影响**：无法实现Rokid设备的手势识别、头部追踪、空间定位等AR功能
**解决方法**：
- **首选方案**：通过Rokid Scoped Registry (npm.rokid.com/) 安装UXR 2.0 SDK包
  - 注意：需验证Unity 6兼容性（官方支持范围是Unity 2022 LTS）
- **备选方案**：采用OpenXR标准插件 + Rokid OpenXR Plugin (com.rokid.xr.unity)
  - 牺牲部分Rokid专属功能（如26骨骼点追踪），换取Unity 6兼容性
- **测试策略**：先在PC端用OpenXR模拟器调试，再部署真机验证

### 缺少条件3：EEG数据采集硬件与接口
**现状**：脑电波采集设备尚未采购，EEG数据读取接口未开发
**影响**：无法实现"脑电信号→AI分析→AR交互"的核心创新点
**解决方法**：
- **硬件选择**：调研并采购消费级EEG设备（如Muse、Emotiv Epoc、OpenBCI等），预算约3000-8000元
- **接口开发**：
  - 学习目标设备的SDK/API文档（通常提供Python/C++/Java接口）
  - 通过Unity Native Plugin或TCP/Socket桥接方式将EEG数据传入Unity
  - 参考开源项目：OpenBCI-GUI、Muse-LSL等已有实现
- **备选方案**：初期使用模拟数据（正弦波+噪声）开发AI分析管线，后期接入真实设备

### 缺少条件4：移动端性能优化经验
**现状**：团队熟悉PC端开发，但缺乏移动端/嵌入式GPU优化实战经验
**影响**：5GB+原始资产无法直接在Rokid AR Lite上运行（可能<10FPS甚至崩溃）
**解决方法**：
- **学习阶段**（第1-2月）：
  - 研读Unity Mobile Optimization官方指南
  - 学习Profiler工具使用（Rendering Profiler、Memory Profiler）
  - 参考案例：其他成功的移动端AR项目优化实践
- **实践阶段**（第9-10月）：
  - 使用Blender Decimate Modifier批量减面（56K→8-12K面）
  - 采用Python脚本批量降分辨率（8K→1024/2048）+ ASTC压缩
  - 使用Unity Addressables实现按需加载
  - 启用SRP Batching和GPU Instancing减少Draw Call
- **迭代优化**：真机profiling → 瓶颈定位 → 针对性优化 → 再测试循环

### 缺少条件5：AI算法训练数据集
**现状**：情绪分类CNN和注意力预测LSTM模型需要标注好的EEG训练数据
**影响**：无法训练准确的AI模型进行实时脑电信号分析
**解决方法**：
- **公开数据集**：利用DEAP、SEED、MAHNOB-HCI等公开情感计算数据集预训练模型
- **自建数据集**：项目中期组织小规模采集（10-20名受试者，观看不同情绪诱发视频时记录EEG）
- **迁移学习**：在大规模公开数据上预训练，再在小规模自采数据上微调（fine-tuning）
- **合作可能**：联系本校心理学院/医学院是否有现成的EEG实验数据和设备可共享

    ```python
    img.save('moon_02_diff_optimized.png', quality=85)
    ```
  - Unity Import Settings修改：Max Size=1024/2048, Format=ASTC 4x4
- 第4周：图集合并（Atlas Generation）
  - 使用TexturePacker或Unity Sprite Atlas工具
  - 将30+张独立纹理合并为4-6张大图集（按类型分组：地形/岩石/角色/特效）

| Rokid UXR 2.0不支持Unity 6 | 中 | 高 | 立即切换OpenXR备选方案（第2个月决策点） |

| 团队成员时间冲突导致进度延误 | 中 | 中 | 提前识别关键路径任务、设置缓冲周、必要时缩减非核心功能范围 |
